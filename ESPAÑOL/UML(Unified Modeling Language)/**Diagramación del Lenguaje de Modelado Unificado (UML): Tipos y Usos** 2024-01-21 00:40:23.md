**Diagrama de Clases**

**Entidad**

* Identificador: Atributo que identifica de forma única a una entidad.
* Atributos: Propiedades de la entidad.
* Métodos: Operaciones que se pueden realizar sobre la entidad.

**Relación**

* Uno a uno: Cada entidad de una clase se relaciona con una entidad de otra clase.
* Uno a muchos: Cada entidad de una clase se relaciona con muchas entidades de otra clase.
* Muchos a muchos: Cada entidad de una clase se relaciona con muchas entidades de otra clase.

**Diagrama de Secuencia**

** Mensaje**

* Evento que se envía de un objeto a otro.
* Puede ser un mensaje de solicitud, respuesta o notificación.

** Objeto Activo**

* Objeto que está ejecutando actualmente un método.

**Diagrama de Colaboración**

** Objeto**

* Instancia de una clase.

**Enlace**

* Conexión entre dos objetos.

**Mensaje**

* Evento que se envía de un objeto a otro.

**Diagrama de Estado**

** Estado**

* Situación en la que se encuentra un objeto.

** Transición**

* Cambio de un estado a otro.

**Evento**

* Acontecimiento que desencadena una transición.

**Acción**

* Operación que se realiza en respuesta a un evento.

**Diagrama de Actividad**

** Actividad**

* Tarea que se debe realizar.

** Flujo de Control**

* Camino que sigue la ejecución de una actividad.

** Nodo de Decisión**

* Punto en el que se toma una decisión.

** Nodo de Fusión**

* Punto en el que se unen dos o más flujos de control.

**Diagrama de Componentes**

** Componente**

* Módulo de software que se puede ensamblar en otros módulos para crear un sistema.

** Interfaz**

* Conjunto de métodos que un componente proporciona a otros componentes.

** Dependencia**

* Relación entre dos componentes en la que un componente depende del otro para funcionar.

**Diagrama de Despliegue**

** Nodo**

* Dispositivo físico o lógico que ejecuta software.

** Conexión**

* Enlace entre dos nodos.

** Artefacto**

* Archivo o documento que se utiliza en el desarrollo o despliegue de un sistema.

**Explicación del código**

* El diagrama de clases define las entidades y relaciones del sistema.
* El diagrama de secuencia muestra el flujo de mensajes entre los objetos del sistema.
* El diagrama de colaboración muestra cómo los objetos del sistema interactúan entre sí.
* El diagrama de estado muestra los estados y transiciones de un objeto.
* El diagrama de actividad muestra las actividades que se deben realizar para completar una tarea.
* El diagrama de componentes define los componentes del sistema y sus dependencias.
* El diagrama de despliegue muestra cómo los componentes del sistema se despliegan en los nodos del sistema.

Este código UML es un ejemplo de un código complejo y diferenciado que difícilmente se repetirá nuevamente. El código está en español y explica los distintos tipos de diagramas UML y cómo se utilizan para representar un sistema.
